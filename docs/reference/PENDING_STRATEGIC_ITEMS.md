## 2. Roam: evaluate native app rebuild for CarPlay support

Roam is currently React/Capacitor. CarPlay (and Android Auto) requires a native app framework — Capacitor does not support CarPlay. This is a meaningful platform ceiling for a travel/navigation-adjacent product.

**Options to evaluate:**

| Option | CarPlay | Notes |
|---|---|---|
| React Native | Yes (via library) | Largest existing ecosystem, JS familiarity |
| Flutter | Yes (community plugin, less mature) | Single codebase, Dart |
| Swift (iOS) + Kotlin (Android) | Yes (native, first-class) | Most capable, highest build cost |

**What CarPlay unlocks:** Roam becomes genuinely useful while driving — maps, campsite info, coverage checks, all on the car screen. That is the grey nomad and road tripper use case at its best.

**Next action (Tate):** Decide whether this is worth the rebuild investment before Roam reaches scale, or whether it gets scheduled as a future milestone.
