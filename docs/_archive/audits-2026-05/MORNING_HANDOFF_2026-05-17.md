## What was built tonight (deeper detail if you want it)

### SMS thread continuity (Option 1)

`src/routes/smsWebhook.js`:
- `loadSmsThread(phone)` reads `kv_store.cowork.sms_thread.<phone>` (4h stale window, capped at 10 exchanges)
- `appendInboundToThread(phone, body, at)` writes the inbound message to the thread BEFORE firing the reflex
- `buildReflexPrompt(...)` now embeds prior exchanges as a `[Prior thread:]` block in the prompt
- The chat session that handles the new SMS is instructed (via prompt) to also append its reply to the same kv_store key so the NEXT SMS sees both sides of the conversation

Thread structure stored in kv_store as text-encoded JSON:
```json
{
  "exchanges": [
  {"from": "tate", "body": "hey", "at": "2026-05-17T05:00:00Z"},
  {"from": "reply", "body": "morning", "at": "2026-05-17T05:00:08Z"}
  ],
  "last_at": "2026-05-17T05:00:08Z",
  "phone": "+61404247153"
}
```

Stale window = 4h. After that, a new SMS is treated as a cold-start (new thread; old one preserved as historical Episode if the chat writes one).

### Telegram bot substrate

`src/routes/webhooks/telegram-bot.js`:
- POST `/api/webhooks/telegram/:secret`  -  URL-path secret + optional `X-Telegram-Bot-Api-Secret-Token` header check
- Allowlist via `kv_store.creds.telegram_bot.allowed_user_ids` (just you for V1)
- Each inbound update fires `reflex.fire` on Corazon (same path SMS uses) with the prompt embedding:
  - chat_id (for reply targeting)
  - prior thread context (`kv_store.cowork.telegram_thread.<chat_id>`, 24h window)
  - the curl command to reply via Telegram Bot API sendMessage
  - instruction to append the reply to the thread after sending
- Markdown formatting OK in replies (unlike SMS which is plain text)
- No length cap (unlike SMS 160 chars GSM)
- Photos / voice / files supported at the Telegram protocol level (V1 ignores non-text  -  extend in V2)

`scripts/setup-telegram-bot.js`:
- One-time bot wiring helper (see Step 2c above)

`src/app.js`:
- Mount: `app.use('/api/webhooks/telegram', require('./routes/webhooks/telegram-bot'))`
- Public path allowlist: `/^\/api\/webhooks\/(stripe|vercel|telegram)(\/|$)/` (auth inside route)
