## Config decisions (with reasoning)

### capacitor.config.json
- `server.url = "https://admin.ecodia.au"` ŌĆö live-url architecture per brief; admin.ecodia.au returns HTTP 200 directly (verified 2026-05-06).
- `server.cleartext = false` ŌĆö admin.ecodia.au is HTTPS; no cleartext fallback needed.
- `server.allowNavigation = ["admin.ecodia.au", "*.ecodia.au", "api.admin.ecodia.au"]` ŌĆö wildcard `*.ecodia.au` covers any future subdomain (auth.ecodia.au, ops.ecodia.au) without rebuild; explicit api host kept for clarity.
- `ios.contentInset = "always"` ŌĆö admin UI manages its own header; "always" inset prevents notch/Dynamic Island clash.
- `ios.scrollEnabled = true` ŌĆö explicit per brief (default is true; making it explicit avoids future Capacitor default drift).
- `ios.limitsNavigationsToAppBoundDomains = true` ŌĆö REQUIRED for persistent cookies + service workers in modern WKWebView. Without this, iOS treats third-party cookies as ephemeral; Tate would log in every launch.

### Info.plist
- `WKAppBoundDomains` = `[admin.ecodia.au, api.admin.ecodia.au, ecodia.au]` ŌĆö pairs with `limitsNavigationsToAppBoundDomains=true` to grant cookie-persistence + service-worker support to these domains. Up to 10 entries allowed by Apple; we use 3.
- `UIStatusBarStyle = UIStatusBarStyleLightContent` ŌĆö admin UI is dark themed; light-content gives white text/icons on the status bar.
- `UIViewControllerBasedStatusBarAppearance = false` ŌĆö global status-bar style override (pairs with UIStatusBarStyle); without this, each VC would need to declare style.
- `UIRequiresFullScreen = true` ŌĆö disables iPad split view and Slide Over; admin UI is single-pane, this avoids odd layouts.
- `CFBundleDisplayName = EcodiaOS` ŌĆö home-screen label.
- `LSApplicationQueriesSchemes = [tel, mailto, sms]` ŌĆö allows the WebView UI to open phone-call / email / SMS handlers when admin UI links to them. Without these, modern iOS silently blocks `tel:` etc.
- `NSAppTransportSecurity` left at default ŌĆö admin.ecodia.au is HTTPS, no exemptions needed.

### project.pbxproj
- `MARKETING_VERSION = 0.1.0` ŌĆö bumped from default 1.0 per brief (Capacitor scaffolds with 1.0).
- `CURRENT_PROJECT_VERSION = 1` ŌĆö Capacitor default; matches brief's "Build: 1".
- `PRODUCT_BUNDLE_IDENTIFIER = au.ecodia.os.mobile` ŌĆö set by `cap add ios` from `cap init` appId.
- `IPHONEOS_DEPLOYMENT_TARGET = 14.0` ŌĆö Capacitor 7 default; sufficient for modern WKWebView features (WKAppBoundDomains needs iOS 14+).
- `TARGETED_DEVICE_FAMILY = 1,2` ŌĆö iPhone + iPad universal. Brief did not exclude iPad explicitly; UIRequiresFullScreen handles split-view concerns.

### Capacitor version pin (NOT a brief item, but a decision required by the env)
- Pinned to `@capacitor/core@^7.6.2`, `@capacitor/ios@^7.6.2`, `@capacitor/cli@^7.6.2`.
- Capacitor 8.x requires Node `>=22.0.0`. VPS runs Node `v20.20.2`. Installing 8.x silently skipped the CLI (`@capacitor/cli` listed in package.json but never landed in node_modules ŌĆö `EBADENGINE` warn only).
- Capacitor 7 supports Node 18+ and is in active maintenance; functionally equivalent for V1 wrapper.
- **Worker 4 must use the same Node version on SY094.** If SY094 is on Node 22+, npm will likely upgrade to 8.x on `npm install` ŌĆö recommended: pin engines in `package.json` to `>=18 <22` later, or pin Capacitor 7 explicitly via `~7.6.2` if 8.x compat is later required.

### NODE_ENV gotcha (encountered + resolved)
- VPS env had `NODE_ENV=production`, which silently skips devDependencies on `npm install`. CLI is a devDep. Resolved by re-installing under `NODE_ENV=development`. README does not surface this since SY094 build env will likely default to development; but logging here so Worker 4 knows the trap.
