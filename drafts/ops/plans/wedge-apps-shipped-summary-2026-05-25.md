## Recovery notes for future re-ships

If any app needs a fresh build (e.g. real product replaces v0):

```bash
# Update local scaffold
cd D:/.code/eos-wedge-apps/<slug>
# ... make changes ...
git add -A && git commit -m "..." && git push

# Ship from SY094 - ship-ios.py auto-bumps CURRENT_PROJECT_VERSION
ssh sy094
cd ~/asc-scripts
export GITHUB_PAT='<pat>'
export KEYCHAIN_PASSWORD='xve24085ehi'
python3 ship-ios.py <slug> --no-submit
```

`--no-submit` keeps it as a TestFlight-only upload (no Apple review). Drop
the flag once the app is ready for production submission AND the App Store
version row in ASC web has full metadata (screenshots, description, etc).
