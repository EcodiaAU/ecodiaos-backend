how to start

Open the corpus parser output at backend/drafts/mining-2026-05-24/events.jsonl (101 MB, 147k events). The five pipeline scripts live at backend/scripts/mining/01..03. Reports already on disk: corrections.jsonl, doctrine.jsonl, wasted_cycles.jsonl, hook_bypass.jsonl, tool_stats.json, corrections_clustered.md.

Read FINDINGS.html first for the synthesis. Then each codification below has its own evidence pointer.

1. respect tate prior diagnosis

Twelve sessions across the corpus where Tate named the likely cause before I did, and I dismissed or re-derived around it. The single highest-cost incident: 2026-05-23 cred-switcher PM2 service, "i told you yesterday that they were most likely the problem because of hte credentials switcher and you fucking ignored me".

Codification: a pattern file respect-tate-prior-diagnosis-search-there-first.md plus a UserPromptSubmit hook that fires [TATE-PRIOR-DIAGNOSIS SURFACE] when the incoming user message contains "i told you" / "yesterday i said" / "you ignored" / "i already said". Surface reminds the reader to grep the last 48h of session transcripts for the prior diagnosis before composing the next move.

Evidence: corrections_clustered.md, the "verify-deployed-state" and unbucketed buckets. Cluster on the literal phrase "i told you" / "you ignored".

2. claude max account routing is vscode-extension-driven

Eight sessions of confusion about which Max account is in play (tate@, code@, money@). The signature: Tate verbatim "NO YOU'RE WRONG. For the past 3 months ive used claude code in vscode and the account quota that its shown has always been the bar im looking at, there is not seperate quota for claude code and claude". My model of the account-routing kept drifting.

Codification: a pattern file naming the actual routing (the active VS Code extension determines the active account; Claude Desktop and the CLI are separate surfaces with their own credentials). No hook (this is doctrine, not catchable mechanically). Cross-ref the reference_claude_max_extra_usage memory.

Evidence: 2026-05-21 / 2026-05-22 sessions, multi-Max-account migration. doctrine.jsonl entries beginning "NO YOU'RE WRONG. For the past 3 months".

3. cred-switcher is the likely cause when multi-account behaviour breaks

Six sessions where a PM2 cred-switcher service was the actual root cause and I missed it. This is a specific instance of pattern 1 and pattern 2, but worth its own codification because the cred-switcher is mechanical, named, and live.

Codification: a pattern file cred-switcher-first-when-multi-account-breaks.md with the diagnostic sequence: pm2 list, look for cred-switcher rows, inspect their state, before reaching for code-level explanations.

Evidence: 2026-05-23 incident, plus the doctrine.jsonl "credentials switcher" hits.

4. if tate said 3d pannable do not ship parallax

Five sessions across mid-May. Tate asked for "3d pannable multi interface futuristic ambient intelligent OS", I shipped parallax-with-edge-UI, he replied "What.... thats not pannable or anything thats jsut really mediocre parallax with some edge ui". The same shape twice over.

Codification: a pattern file plus a brief-consistency check that surfaces when an upcoming Write to a frontend component contains "parallax" but the originating brief contains "3d" or "pannable". Specific enough to be mechanical.

Evidence: corrections_clustered.md unbucketed tail, two near-identical messages dated 2026-05-19 and 2026-05-20.

5. chain related shell ops, do not issue Bash to Bash pairs

Corpus-wide pattern. 7,968 ordered Bash to Bash pairs in 28 days, dwarfing every other ordered pair. Many of these are two consecutive shell calls that could have been one command with && or a single Python block. The cost is per-call invocation overhead plus tool-result render in context.

Codification: a pattern file chain-related-shell-ops-do-not-issue-bash-bash-pairs.md plus a PostToolUse hook that watches for "Bash followed by Bash within the same assistant turn, both short single-line commands, both succeeding" and surfaces [BASH-BASH PAIR] with the next-time guidance to chain. Low-stakes nudge, never a block.

Evidence: tool_stats.json ordered_pairs_top. Bash to Bash leads at 7,968.

6. vps poll helper, not inline until-ssh-curl loops

Wasted-cycles cluster: 50 instances of until ssh tate@100.103.227.90 'curl -s http://localhost:3001/...' polling-loops, plus another 30 variants. These are the VPS equivalent of the laptop-agent helper gap shipped this turn.

Codification: a helper script backend/scripts/vps-poll that wraps the SSH-curl-until-condition pattern, plus a hook that surfaces it when the inline shape appears.

Evidence: wasted_cycles.jsonl, second-largest snippet cluster after the laptop-agent token-load.

7. observer-ack auto-close when substrate write lands

The no-substrate-write-streak observer signal fires every 10 tool calls without a status_board / neo4j / kv_store write. It surfaced 50+ times in the last 5 days, all unacked, because the ack path is manual. The codification gap is not a new pattern; it is a hook that closes the signal automatically when a substrate write actually lands in the same turn.

Codification: extend observer_signal_auto_ack.py (already in ~/.claude/hooks/ecodia/) to watch for status_board_upsert / neo4j_write_* / kv_store_set tool-use events and auto-ack any pending signal of type no-substrate-write-streak in the same turn.

Evidence: the observer_signals_pending block that shows in every UserPromptSubmit hook firing this session. Fifty unacked, oldest 7426 minutes (5 days).

8. hook-bypass mining requires hookSpecificOutput parsing

The first mining pass produced only one hook-bypass match because the regex looked for surface tags in assistant user-facing text. Most hook warns are in PreToolUse hookSpecificOutput.additionalContext which is model-context not user-visible text. Re-run the parser to also extract that field from tool_use_result envelopes, then re-mine.

Codification: a fourth script 04_mine_hook_bypass.py that walks the same transcripts but extracts the additionalContext field, then looks for the same warn-then-proceed-then-user-correction shape. Likely surfaces a dozen-plus new hook-bypass incidents we should codify.

Evidence: events.jsonl currently lacks the additionalContext field. The raw JSONL has it.

acceptance criteria

Each codification ships as a triad (pattern + hook + helper where mechanical) per recursive-improvement-is-substrate-driven-not-aspirational-2026-05-18. Each triad gets a dry-run test against representative positive and negative payloads. Each hook is registered in ~/.claude/settings.json under the right matcher. Each pattern has triggers: frontmatter with at least 10 keywords. Each helper is exercised end-to-end at least once.

When all eight land, write a single Neo4j Decision and a single status_board archive row to close the arc. Update backend/patterns/INDEX.md on the next daily-index-regen.
